主版本号.次版本号.补丁号
- jajor 表示主版本号。表示大版本，这个版本更新是颠覆性的修改，直接重构或其他。此时需要更改主版本号。
- minor 表示次版本号。表示大版本大的方向没变，向里添加新的功能或修改部分功能时。此时升级这个次版本号即可。
- patch 表示补丁号。表示向此版本中添加了一些补丁，很小的修改，如有个bug需要修复等。此时升级这个补丁号即可。一般 偶数表示稳定的 patch，基数表示步稳定的patch

包配置版本时，一些符号的意思：
如package.json中依赖包的配置：
"dependencies":{
    "jquery":"^1.12.4"
}

^       这个符号表示，这个配置只锁定”主版本号“。如，"jquery":"^2.2.1" ,当执行 npm update 时，会更新到 2.x.x 的最新版本。。
~       锁定主版本和次版本。如，"jquery":"~2.2.1" ,当执行 npm update 时，会更新到 2.2.x 的最新版本。
*       表示如果更新，更新到最高版本。如 "jquery":"*" ,当执行 npm update 时，会更新到该包的最新版本。
不带符号 表示就使用该版本，当执行 npm update 时，不会更新。
