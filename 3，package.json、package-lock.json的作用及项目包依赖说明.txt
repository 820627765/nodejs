1，package.json 的作用？
- 记录了项目安装了哪些包。
- 当我们项目提交代码仓库或拷贝给同事时，就不需要将 node_modules 一起提交或拷贝（因为它很大）。因为package.json 记录了项目依赖了
  哪些包，所以只需要拷贝这个 package.json 即可。拿到代码的人只需要执行 npm i  接口安装package.json 中记录的所有包。
  注意1：npm i 是表示安装package.json 中记录的所有包。
  注意2：npm 安装一个包后，你会发现 node_modules 下有特别多的包，实际上是因为你安装的包也会依赖其他包，所以会将其所依赖的包一起安装。

2，package-lock.json 的作用？
- 将每个包所依赖的依赖关系都记录下来。记录的内容很庞大，也很难看，可以使用 npm list 来显示项目下的包的依赖关系。

1，项目中package.json 就是包依赖的配置文件。
2，不同环境的包存在不一样，所以package.json 中也区分环境，如开发环境的依赖，生产环境的包依赖
   dependencies：{} 中配置的是生产环境需要的包依赖。使用 npm i 包名 -S  就会将包信息配置到该节点下
   devDependencies:{} 中配置的是开发环境需要的包依赖。使用 npm i 包名 -D  就会将包信息配置到该节点下

   注意：如果 npm 安装的是一个 git仓库的资源，则package.json 中的依赖会是一个git地址。
